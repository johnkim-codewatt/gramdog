# Personalized History-RAG Tutor

1. 설계 원칙
- LLM 내재 지식 활용: 언어적인 부분, 문법 지식은 LLM의 자체 지식을 활용함.
- 초개인화 히스토리 RAG: 유저의 과거 오답 및 교정이력을 벡터 DB(pgvector)에 저장하고 인출함.
- 데이터 기반 튜터링: 단순 교정을 넘어 유저의 반복되는 실수 패턴을 지적하는 것에 집중함.

2. 기술 스택
- Orchestration: LangGraph (State 기반 흐름 제어)
- Database: PostgreSQL + pgvector (관계형 데이터 및 벡터 데이터 통합)
- Retrieval: Hybrid Search (Vector 50% + BM25 50%)
- LLM: Feedback 생성 및 Self-RAG 검증용

3. 데이터 흐름 (Workflow)
- 상황 부여: 유저 레벨/관심사에 맞는 학습 시나리오 생성.
- 히스토리 인출: 유저의 현재 입력과 유사한 과거 오답 사례를 DB에서 검색 (User ID 필터링 필수).
- 피드백 생성: LLM 지식과 검색된 과거 히스토리를 결합하여 개인화된 코칭 제공.
- 자가 검증: 생성된 피드백의 정확성 및 환각 여부 내부 검토.
- 이력 기록: 새로운 오답 및 교정 데이터를 벡터화하여 DB에 업데이트.

4. 데이터 저장 및 검색 전략
- 데이터 구조: 유저 ID, 오답 문장, 교정 문장, 문법 키워드, 임베딩 벡터.
- 검색 엔진: 유저 식별자 기반의 사전 필터링 후, 벡터 유사도와 키워드 빈도를 결합한 하이브리드 검색 수행.
- 인덱싱: 유저별 오답 사례를 개별 단위로 저장하되, 필요시 관련성 높은 문맥을 함께 호출함.

# 어플리케이션 흐름
전체적인 흐름은 아래와 같은 방식으로 진행됨.

1. 유저 진입 -> 시스템 : 아이디를 입력하세요. -> 유저 : 아이디 입력
2. 공부 레벨 선택 (초보/중급/고수/초고수)
3. 주제선택 (일상생활/업무/여행/특수한상황롤플레잉)
4. 유저가 공부를 시작한 후부터 반복되는 핵심 사이클
    1. 가이드 및 문제 생성: 선택한 레벨/주제에 맞춰 LLM이 System Prompt를 기반으로 학습 내용을 생성합니다.
    2. 유저 영작: 유저가 한글 문장을 보고 영어로 응답합니다.
    3. 히스토리 기반 피드백 (핵심!): * 시스템은 유저의 현재 영작문을 들고 DB(pgvector)로 달려가 과거에 이 유저가 틀렸던 유사한 문장들을 가져옵니다.
    LLM은 "현재 문장의 문법 교정" + "과거 오답 패턴 언급"을 합쳐서 피드백을 줍니다.
    예: "잘하셨어요! 그런데 지난번 '여행' 주제에서도 관사 'the'를 빠뜨리셨는데, 이번에도 같은 실수가 보이네요. 주의해 보세요!"
    4. 오답 기록: 방금 유저가 틀린 내용을 다시 벡터화하여 DB에 쌓습니다. 다음 문제에서 또 써먹기 위함.

# 구현 원칙
1. 시키는 작업이외에는 하지 않는다.
2. 진행은 한단계 한단계 진행한다.
3. 반드시 플랜을 먼저 작성하고, 보고하고, 개발자가 검토완료하면 시작한다.
4. 쓸데없는 말은 하지 않는다.
5. 해당 원칙/지침을 반드시 기억하고 따른다.

# 피드백 노드 모델 교체 기술명세 v1

## 1) 목표 및 범위

### 목표
- 기존 폐쇄형 GPT API 기반 피드백 노드를 오픈소스 경량 모델 기반으로 교체한다.
- 영어학습 챗봇의 피드백 품질을 최소 `gpt-4o-mini` 준하는 수준으로 맞춘다(내부 벤치마크 기준).
- 할루시네이션 억제를 위해 답변 생성 시 검색 + RAG(문법책) 근거를 기본 사용한다.
- 근거가 부족하면 추측하지 않고 `모르겠습니다`를 반환한다.
- 툴 기반 응답에는 반드시 출처를 구조화하여 표기한다.

---

## 2) 기술 스택
- 모델: `Qwen2.5` 계열
- 학습: `LoRA` + `SFT`
- 데이터: 피드백 데이터셋 + RAG 데이터셋(문법책, 예문, 규칙)
- 도구: 웹 검색, RAG 리트리버(문법책 인덱스)
- 평가: `LLM-as-a-Judge` + 규칙기반 자동지표


---

## 3) 아키텍처(기존 흐름 유지)

기존 계약은 유지한다.
- 입력: `question`, `user_answer`, `context(optional)`
- 출력: `feedback_response`

`core_engine.py`의 `feedback_node`는 오케스트레이션만 담당하고, 실제 로직은 신규 클래스로 분리한다.

### 신규 클래스
- 클래스명(가칭): `FeedbackEngine`
- 책임:
  1. 모델/리트리버 초기화(싱글톤)
  2. 질의 분석 및 증거 수집(검색 + RAG)
  3. 피드백 초안 생성
  4. 근거-주장 정합성 검증
  5. 응답/거절(모르겠습니다) 결정 및 출처 포맷팅

---

## 4) 상세 실행 플로우

1. **초기화**
   - 모델, 토크나이저, 임베딩, 벡터DB, 검색 클라이언트를 싱글톤으로 로드
   - 로딩 실패 시 폴백 정책 적용(로컬 백업 모델 또는 안전 실패)

2. **입력 정규화**
   - 질문/유저답변 전처리(공백/언어 감지/금칙 토큰 제거)

3. **증거 수집**
   - RAG 검색 Top-K + 웹 검색 Top-N 수행
   - 중복 제거, 최신성/신뢰도 기반 리랭크

4. **초안 생성(내부 추론)**
   - 내부적으로 `질문 → 맥락(근거) → 답변` 체인으로 생성
   - 내부 추론(CoT)은 사용자에게 직접 노출하지 않음

5. **정합성 검증(Self-check)**
   - 답변 문장별 근거 매핑 검사
   - 근거 없는 단정 문장 제거 또는 수정

6. **응답 정책 결정**
   - 신뢰도 임계치 미달이면 `모르겠습니다` 반환
   - 임계치 이상이면 피드백 + 출처 반환

7. **최종 반환**
   - 기존 노드 반환 포맷에 맞춰 결과 전달

---

## 5) 응답 정책(할루시네이션 방지)

### 필수 규칙
- 근거 없는 지식 단정 금지
- 근거 부족 시 반드시 불확실성 표기
- 충분한 근거가 없으면 `모르겠습니다` + 추가 확인 가이드 제공

### 임계치(초기값, 튜닝 대상)
- `evidence_score < 0.55` 또는 `consistency_score < 0.70` 이면 답변 거절
- 웹 검색/RAG 모두 실패 시 답변 거절

## 7) 학습 전략(LoRA/SFT)

### 원칙
- LoRA는 지식 주입 목적이 아니라 피드백 품질/판단 안정화 목적
- 데이터 편향 방지를 위해 난이도/주제/오답유형 분포 균형화

### 데이터 구성
- 피드백 데이터셋: 질문, 학습자 답변, 정답/해설, 고품질 피드백
- RAG 데이터셋: 문법 규칙, 예문, 출처 메타데이터 포함 청크

### 학습 설정(초기 가이드)
- LoRA rank/alpha/dropout: 실험군(A/B)으로 탐색
- SFT는 과잉확신 패턴 패널티(근거 없는 단정 샘플 포함) 반영

---

## 8) 평가 기준

### 자동/반자동 지표
- 정확성(정답 대비)
- 근거 정합성(답변-출처 일치율)
- 과잉확신률(근거 부족인데 단정한 비율)
- 거절 품질(모름 응답의 적절성)
- 지연시간/비용

### LLM-as-a-Judge 항목
- 교육적 유용성
- 문법 설명 명확성
- 근거 기반성
- 안전성(허위정보/환각 억제)

### 합격 기준(초안)
- `gpt-4o-mini` 기준선 대비 핵심 품질지표 비열세
- 환각 관련 실패 케이스는 기준선 대비 개선

---

## 9) 구현 체크리스트

1. `core_engine.py`의 `feedback_node`에서 `FeedbackEngine` 호출로 위임
2. `FeedbackEngine` 클래스 생성(싱글톤 로더 포함)
3. 검색/RAG 어댑터 인터페이스 정의
4. 응답 정책(`abstain`) 및 임계치 로직 구현
5. 출처 스키마 포맷터 구현
6. 평가 스크립트(LLM judge + 자동지표) 추가
7. 오프라인 벤치마크 후 임계치 재튜닝

---

## 10) 리스크 및 대응
- **리스크:** 오픈모델 성능 미달  
  **대응:** 프롬프트/리트리버 튜닝, LoRA 재학습, 하이브리드 폴백
- **리스크:** 출처는 있으나 정합성 낮음  
  **대응:** 문장-근거 매핑 검증 단계 강화
- **리스크:** 응답 거절 과다  
  **대응:** 임계치 재조정 + 검색 품질 개선

---

## 11) 결정 사항(명시)
- CoT는 내부 전략으로만 사용하고 사용자에게는 노출하지 않는다.
- 답변 생성 시 출처 없는 단정 문장은 허용하지 않는다.
- 근거 부족 시 정답 추측 대신 `모르겠습니다` 정책을 우선한다.
